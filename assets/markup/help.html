Help

I get 'bad value' when designing my coin.

Build-a-Coin tries to cut down on compile errors by rejecting inputs that would break your new coin. Limitations on input values are as follows:

  • All strings must contain only ASCII printable characters.
  • All string values must not contain quotes ["'] nor backslashes [\].
  • All strings must be no more than 256 bytes long
  • The name and currency code must consist only alphabetical characters [a-zA-Z].
  • Numeric values must fit into the datatype they will be in your coin's code. Errors in numeric fields probably mean they're too big or negative.
  • Pubkeys must be 65 bytes expressed in hexadecimal (130 hex digits)

My coin won't compile.

Build-a-Coin attempts to always produce correct code. If you haven't edited your code at all and have ruled out a build system misconfiguration, please email info+buildacoin@shick.xyz with the following:

  • your coin's build-a-coin.txt or coin identifier
  • operating system version
  • compiler version
It is the policy of Build-a-Coin that if a coin compiles on our build platform (Debian/gcc), then it is working. Although Build-a-Coin coins should compile anywhere bitcoin can be compiled, we do not guarantee this.

My coin doesn't work right.

If you put unreasonable values in, you'll get an unreasonable coin out. Build-a-Coin coins have been tested for correctness with reasonable inputs, but are not guaranteed to work. If you believe you've found a bug in Build-a-Coin itself, report it to info+buildacoin@shick.xyz.

I don't know how to compile my coin.

Build-a-Coin does not provide any support for the compilation process; there are many guides freely available that cover this topic.
